# Recall In-App Updates

Recall uses the official Tauri v2 updater plugin with a signed static JSON manifest. The app checks the configured manifest endpoint, verifies update signatures, downloads the update package, installs it, and relaunches.

## Current Configuration

The updater endpoint is configured in `src-tauri/tauri.conf.json`:

```json
{
  "plugins": {
    "updater": {
      "endpoints": ["https://raw.githubusercontent.com/siddhantsrivastava7-pixel/recall/main/updates/latest.json"],
      "windows": {
        "installMode": "passive"
      }
    }
  }
}
```

`bundle.createUpdaterArtifacts` is enabled so release builds can generate updater artifacts alongside the installer.

## Static Manifest Shape

Host a JSON file at the configured endpoint using this shape:

```json
{
  "version": "0.2.0",
  "notes": "Improved capture reliability and bookmark search.",
  "pub_date": "2026-04-10T12:00:00Z",
  "platforms": {
    "windows-x86_64": {
      "signature": "PASTE_GENERATED_SIGNATURE_HERE",
      "url": "https://github.com/siddhantsrivastava7-pixel/recall/releases/download/v0.2.0/Recall_0.2.0_x64_en-US.msi"
    }
  }
}
```

The exact artifact filename depends on the output generated by `tauri build`. Use the signed updater artifact and matching signature generated for each release.

## Signing

Updater artifacts must be signed. For the current local development keypair:

```powershell
$env:TAURI_SIGNING_PRIVATE_KEY=Get-Content -Raw "C:\Users\siddh\OneDrive\Desktop\recall\imported\recall-tauri-full-4\recall-updater-private.key"
$env:TAURI_SIGNING_PRIVATE_KEY_PASSWORD="recall-dev-updater-key"
npm.cmd run tauri:build
```

This generated key is suitable for local development only. For production, generate and store the private key in a secure release environment, publish only the public key in `tauri.conf.json`, and never commit the private key.

## Windows Notes

- Windows is currently configured with passive install mode.
- Users can manually check for updates in Settings > Updates.
- Startup checks are optional and never auto-install updates.
- Signature mismatch, unreachable manifests, malformed manifests, and failed downloads are surfaced as calm in-app errors.

## macOS Notes

The service and store are platform-agnostic. Future macOS release support should add the `darwin-aarch64` and/or `darwin-x86_64` entries to the same static manifest and verify macOS signing/notarization requirements in the release pipeline.
